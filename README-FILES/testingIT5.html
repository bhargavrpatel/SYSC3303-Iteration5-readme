    It will be assumed that the project has been imported in Eclipse and setupTestFiles.java has been run at least once. If you have not done so, click this link to learn how. Also, please check out changelog.html.
    Assuming you will be testing on two machines, the first machine (on which you will run the Client) will be referred to as Machine A and the seccond machine (on which you will run the errorSim and Server or simply Server) will be reffered to as Machine B
    Testing With Error Simulator on
    1. Run setupTestFiles.java on Machine 2.
    2. Run Server.java on Machine 2 as per before.
    3. Choose a directory for the server files or simply press enter to use the default (preferred)
    4. Run errorSim.java on Machine 2 as per before.
      5. You may introduce any errors if you wish in the errorSim. The method of doing so has not changed from iteration 4. Simply refer to Iteration 4 page if the menu you are greeted with is confusing.
    5. On Machine 2, visit www.ipchicken.com and take note of the IP address or the Name Address. Like Shown below.
      6. Screenshot of ipchicken

      Click To Enlarge
    6. Run setupTestFiles.java on Machine 1.
    7. Run Client.java on Machine 1.
    8. Choose a directory for the client files or simply press enter to use the default (preferred)
      9. You will notice the Client Menu has changed to show the IP/Name Address and Port on which we will send requests to.
      Screenshot of the "connection status"

      Click To Enlarge
    9. Since we are testing with Error Sim on, in the Client console (on Machine 1) type "normal off" to turn off normal mode. Once you press enter the menu will reprint
      10. Client menu: Turning off Normal mode

      Click To Enlarge
    10. Type "connect" and press enter. You will be greeted with a submenu.
    11. In the submenu, type either the name address or the IP address of Machine 2 (we prefer name address) that you have taken note of.
      12. Client menu: Changing connection address

      Click To Enlarge
    12. You may test Read/Write requests in combination of any possible errors now. You should also try the various special test files we have provided, list of which can be found here
